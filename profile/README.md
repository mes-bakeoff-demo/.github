# 🍰 MES Bake-Off Demo

**An independent demonstration of challenge-based procurement for Medicaid Enterprise Systems**

> ⚠️ **This is not an official government project.** This demo was created by Nick Aretakis, inspired by Minnesota DHS's presentation at the 2026 HIT Connect Conference. It is not affiliated with, endorsed by, or representative of any state government.

---

## What Is This?

This GitHub Organization demonstrates what **transparent, GitHub-powered procurement** could look like for state Medicaid systems. Instead of traditional 18-36 month RFP cycles, we show how states can:

- 📋 **Publish a backlog** of outcomes they need (not requirements they've guessed)
- 🏃 **Run rapid evaluations** in days/weeks, not months/years  
- 🍰 **Let multiple vendors compete** on the same "slice" of work
- 📊 **Evaluate on delivered value**, not written promises

## The Demo Slice: MSP Auto-Enrollment

Minnesota identified **Medicare Savings Program (MSP) eligibility** as their first "slice" — a complete, end-to-end use case that delivers real value.

Three vendors built working prototypes in parallel, each with a different approach:

## Why GitHub?

| Need | GitHub Feature |
|------|----------------|
| Transparent backlog | **Issues** — public, searchable, linkable |
| Vendor Q&A | **Discussions** — threaded, organized |
| Progress tracking | **Projects** — Kanban boards |
| Code review | **Pull Requests** — visible, auditable |
| Deployment | **Actions + Pages** — automated, free |
| Documentation | **Markdown** — version controlled |

## The Philosophy

> "Move decision-making about services and software to AFTER seeing them working in context, instead of spending time deciding BEFORE."

Traditional procurement asks vendors to *promise* what they'll build. Challenge-based procurement asks vendors to *show* what they can build.
